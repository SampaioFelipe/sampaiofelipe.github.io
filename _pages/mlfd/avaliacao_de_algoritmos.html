Avaliação de Algoritmos de Fluxo de Dados

Avaliação do contexto "batch"

O procedimento de avaliação define quais eemplos são usados para treinamento e quais são usados para teste.

  • Conjuntos pequenos (até 1000 exemplos): o objetivo é extrair o máximo dos exmplos

Essa validação ainda é algo novo no contexto de fluxo de dados.

Procedimentos de Avaliação

Holdout

Extensão do processo usado no contexto batch.

Um conunto de exemplos para testes é separado e o modelo corrente é aplicado a esse conjunto a intervalos de tempos regulares.

  • Conjuntos Estacionários
  • Conuntos Não Estacionários

Predictive Sequencial (Presequential)

O erro do modelo é calculado a partir da sequência de exemplos.

Prequential error:

$$ S = \sum_{i=1}^n L(y_i,\bar{y_i}) $$

Comparação entre estimadores de erro usando um algoritmo e um conjunto de dados

A avaliação prequential fornece uma curva que mostra a evolução do processo de aprendizado.

Avaliação holdout fornece uma curva semelhante aplicando o modelo corrente a intervalos regulares no conjunto.

As duas estimaticas podem ser afetadas pela ordem dos exemplos.

Mecanismo de Esquecimento para Prequential

A estimativa de erro usando todo o fluxo pode sofrer influência das primeiras estimativas, quando o modelo foi gerado com poucas instâncias.
