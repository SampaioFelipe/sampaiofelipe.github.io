Lista 1 de Exercícios

1. O que é um Sistema Operacional? Quais são suas funções?

O SO pode ser visto de duas formas diferentes:

  • Como um Gerente de Recursos: responsável por gerenciar a utilizaçao dos recursos (utilização da CPU, acesso a memória, interfaces de rede, I/O, vídeo, e afins) pelos os diversos programas que precisam utilizá-los.
  • Como uma Máquina Virtual: responsável por fornecer abstração dos recursos da máquina, de forma a disponibilizar ao usuário uma interface segura e facilitada para a utlização desses recursos.

De forma geral, o SO é um software (um conjunto de instruções) responsável por facilitar a interação do usuário com os recursos que uma máquina oferece e por gerenciar, da forma mais justa e segura, a utilização desses recursos.

2. Descreva o modo de operação de um computador de acordo com a arquitetura de Von Neumann. 

      LOOP {
        IR <- MEM[PC] // Registrador de Instrução (IR) recebe a instrução apontada pelo Contador de Programa (PC)
        PC <- PC + 1 // PC aponta para a próxima instrução
        EXEC IR // Executa a instrução que está em IR
          INT ou EXCEÇÃO? // tratamento de interrupção ou exceção caso ocorra
      }

Segundo a arquitetura de Von Neumann, a memória de um computador é utilizada para armazenar tanto dados quanto instruções.

3. De que maneira são representados os processos em um sistema operacional?

Os processos são representados como estruturas, no caso do Linux a struct task_struct definida em C, que armazenam informações (como identificador, estado dos registradores, mapeamento de memória, estutura de tratamento de sinais,arquivos abertos pelo processo, status de execução) essenciais para que o SO possa gerenciar processos.

4. O que é contexto de um processo? Como se relaciona com a representação de processos feita pelo SO?

O contexto de um processo é seu estado, ou seja, o conjunto de informações mantidas pela estrutura do processo.

5. Como ocorrem as trocas de contexto?

Após ocorrer um evento de troca de contexto (terminado o time slice do processo ou outro evento interno do SO, como decisão de deixar o processo esperando enquanto o SO está providenciando um recurso pedido) é lançada uma interrupção. A rotina de tratamento salva o estado corrente dos registradores (push cs, push eip, push EFLAGS), o SO então entra em ação, atualizando a estrutura do processo com o seu estado corrente. Determina qual processo ficará no lugar através de alguma polpitica de escalonamento.

Em seguida, carrega o estado do novo processo configurando os registradores do processador (retorna o estado do processado ao momento que estava quando esse processo foi interrompido anteriormente). Com tudo configurado, o SO sai de cena e deixa o processo tomar conta da CPU.

6. Descreva estratégias usadas por sistemas operacionais para o compartilhamento do tempo de execução de um processador entre diversos processos.

Essas estratégias de compartilhamento de tempo são denominadas de políticas de escalonamento. De forma geral, podemos dividir os algoritmos de escalonamento em duas categorias: não-preemptivos e preemptivos.

Algoritmos não-preemptivos esperam que o processo termine sua execução, ou libere a CPU por livre e espontânea vontade, para dar espaço a outro processo. Algoritmos preemptivos, por sua vez, fazem com que o processo deixe a CPU a qualquer momento (de acordo com sua política).

Dentro dessas categorias temos diversas estratégias para a seleção de qual processo será o próximo a ser executado. Essas estratégias levam em conta o propósito do sistema.

7.Descreva técnicas usadas pelo sistema operacional para interagir com os controladores de dispositivos.

A parte do SO responsável por interagir com controlador físico de dispositivo é chamada de device driver, que são softwares escritos pelos fabricantes do dispositivo.

O processador pode se comunicar com esses controladores através do endereçamento de um barramento direto com eles ou pelo endereçamento de espaços na memória reservados a esses dispositivos.

8. O que são interrupções? Descreva os tipos existentes e os eventos aos quais estão associados.

Uma interrupção é uma "notificação" enviada ao processador, que indica que algum evento deve ser tratado.

Existem dois tipos de interrupções: internas, geradas em decorrência da execução de uma instrução, ou externas, geradas por controladores de dispositivos.
