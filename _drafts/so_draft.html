O que é um aquivo?

Do ponto de vista de um processo, o SO mantém um vetor de arquivos abertos. Para cada arquivo há informaçoes sobre:

Para o processo um arquivo é uma sequência de bytes.

Para o SO, um arquivo é metadados (permissões, owner ....) + conjunto de blocos do disco. O SO converte operações em bytes em operações que lidam com o conjunto blocos.

Como funcoina um disco?

O disco é um dipositivo de bloco

O SO ve o disco como alguma geometria, definida pelo controlador de disco.

No processo de formatação do disco, o SO identifica a geometria do disco, cria estruturas para mapeamento de setores com relação aos arquivos/diretórios.

O SO sempre pega o bloco inteiro, para isso aloca paginas para isso (por isso o tamanho de um bloco = tamanho de uma pagina = 4 kb)

A escrita no arquivo é feita on demand. Fica armazeda em algum buffer, dps vai ser gravado de fato.
